## Major changes

- Reduced `SKILL.md` to a navigation-first entrypoint with only the critical rules and read order.
- Moved the phase protocol into dedicated workflow files so each phase can be loaded independently.
- Extracted handoff contracts into a single handoff resource.
- Extracted autogenerated artifact contracts and templates into dedicated template files.
- Split examples and scoring details into separate example and reference files.
- Added a validation checklist to make the preserved constraints explicit.

## Preserved behavior

- The orchestrator remains a coordinator and must not implement code directly.
- Phases 3, 4, 5, and 6 remain mandatory and cannot be skipped.
- Execution still requires plan persistence, progress-file initialization, plan review with the user, and explicit confirmation before phase 4.
- Explorer remains the required path for high-level context in phase 1 and detailed context in phase 2.
- Execution still uses complexity-based coder selection, parallel groups, review after every task, and up to 3 attempts per task.
- The final user closure still happens only in phase 6 through the finalized progress file.
- All handoff headers and contracts were preserved.
